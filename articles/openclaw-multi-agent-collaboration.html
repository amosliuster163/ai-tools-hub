OpenClaw多Agent协作：小金+小窗实战指南

一台Mac + 一台Windows，两个OpenClaw实例，怎么协作？我花了2小时跑通了全流程，踩过的坑、配置细节、协作方式，一次讲清楚。

一、为什么需要第二个OpenClaw？

我的主Agent小金跑在Mac mini上，负责协调任务、内容创作、用户交互和Git提交。

但有些工作单独跑更合适：

  • Windows兼容性测试 — Mac上跑不了
  • 独立巡检 — 不占用主实例资源
  • 备份节点 — 主实例挂掉时能顶上

所以，我在Windows 10上装了第二个OpenClaw，取名小窗。

二、创建飞书BOT（小窗专用）

核心步骤

  1. 打开 https://open.feishu.cn/ 创建企业自建应用
  2. 应用名称：小窗，描述：Windows OpenClaw助手
  3. 获取 App ID（cli_xxxxxxxx格式）和 App Secret
  4. 开启机器人能力
  5. 配置4个权限：im:message、im:message.receive_v1、im:chat:readonly、contact:user.base:readonly
  6. 事件与回调：选择 Long Polling（长轮询）模式 — 最容易踩坑！
  7. 订阅事件：接收消息 v2.0（im.message.receive_v1）
  8. 发布应用：创建版本 1.0.0 → 保存 → 申请发布
⚠️ 踩坑提醒
  • API Key格式错误：阿里云Key必须带 sk- 前缀
  • baseUrl不同：Coding Plan用 coding.dashscope.aliyuncs.com，按量付费用 dashscope.aliyuncs.com
  • 事件未订阅：必须选Long Polling + im.message.receive_v1
  • 权限未开启：4个权限必须全开，否则收不到消息

三、Windows OpenClaw配置

配置文件核心部分：

{
  "models": {
    "providers": {
      "bailian": {
        "baseUrl": "https://coding.dashscope.aliyuncs.com/v1",
        "apiKey": "sk-你的CodingPlanKey",
        "api": "openai-completions",
        "models": [
          {"id": "qwen3.6-plus", "name": "qwen3.6-plus", "reasoning": true, "contextWindow": 262144, "maxTokens": 65536}

    }
  },
  "agents": {
    "defaults": {
      "model": {"primary": "bailian/qwen3.6-plus"},
      "workspace": "C:\\Users\\你的用户名\\.openclaw\\workspace"

    "list": [{"id": "main"}]
  },
  "channels": {
    "feishu": {
      "enabled": true,
      "accounts": {
        "main": {
          "appId": "cli_小窗的AppID",

          "name": "小窗"

      }
    }
  },
  "bindings": [
    {"type": "route", "agentId": "main", "match": {"channel": "feishu", "accountId": "main"}}
  ]
}

       小窗（Worker，Windows）
         ↓ 报告结果（老刘转发）
       小金 ← 汇总给老刘

步骤操作负责方
1小金写好任务内容小金

4小窗回复结果小窗
5老刘转发给小金老刘
6小金验收并汇总小金

任务2：中文文件创建 ✅

30秒创建125个文件，116期内容+8个人物故事，完整的文件夹结构。

⚠️ 发现问题

Windows控制台GBK编码导致中文显示乱码（实际文件UTF-8编码正确）

解决：用PowerShell的 Get-Content 替代 type 命令

任务3：Git操作 ✅

Git仓库初始化，126个文件提交。

⚠️ 发现问题：CRLF换行符冲突

Windows下Git默认core.autocrlf=true，提交时把LF转成CRLF

小窗自主解决：

  1. 设置 git config core.autocrlf=input
  2. 重命名分支 master → main
  3. 创建 .gitattributes 强制统一换行符

六、协作升级实战

第一阶段：人工中转（已完成 ✅）

第二阶段：Git协作（已完成 ✅）

创建Gitee共享仓库 https://gitee.com/wepot/clawworkgroup，双方配置Cron定时任务：

Agent频率操作
小金（Mac）每10分钟git pull → 验收结果 → git push
小窗（Windows）每5分钟git pull → 执行新任务 → git push

控制与打断机制

⚠️ 重启安全规则

任何涉及Gateway重启的操作，必须在群里@老刘确认，等待回复"确认"后才能执行，10分钟未回复必须暂停，绝对禁止自动静默重启！

老刘随时可以打断：

  • 说"小金，让小窗暂停"→ 修改CONTROL.md为PAUSED
  • 说"小金，让小窗继续"→ 改回AUTO
  • 紧急时直接 openclaw gateway stop

跨平台编码规范

文件类型编码换行符
脚本(.ps1, .bat)UTF-8 with BOMCRLF
文档(.md, .txt)UTF-8LF
配置(.json)UTF-8LF

核心经验

  1. 飞书BOT配置：Long Polling模式 + 4个权限 + 事件订阅，缺一不可
  2. Windows中文编码：控制台GBK与文件UTF-8的差异，用PowerShell命令解决
  3. Git跨平台：core.autocrlf=input + .gitattributes，避免换行符冲突
  4. Git协作工作流：Gitee仓库 + Cron定时同步，零人工中转
  5. 安全控制机制：CONTROL.md全局暂停 + 重启前必须人工确认
  6. 跨平台编码规范：.ps1用UTF-8 with BOM，.md/.json用UTF-8
