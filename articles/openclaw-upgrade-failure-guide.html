OpenClaw升级翻车实录

一个npm包更新，让我AI助手瘫痪了3小时

周日早上给OpenClaw升了个级，从v2026.4.2到v2026.4.11。心想升个级而已，能有多难？

结果：升级完，AI助手完全不回复消息了。飞书、微信全部沉默。

排查了3小时，最后发现——一个上游npm包移除了一行代码，导致19个插件全部加载失败。

升级开始

执行 openclaw update，报错：working directory not clean。

解决：git stash -u 后再次升级，跑通。

第一层故障：config验证失败

执行 openclaw gateway restart，报错：config验证失败。

原因：free-quota-chain 的 fallback key不被新版本schema支持。

教训1：旧配置不一定兼容新版本。升级前先检查config。

第二层故障：插件全部加载失败

重启后报错：plugin manifest not found。我手动创建了假的manifest文件——这是第一个大错。

再次重启，报错更严重：normalizeLowercaseStringOrEmpty is not a function。整个系统瘫痪。

教训2：不要伪造manifest文件。直接 git clean -fd 更干净。

根因分析

错误传播链：
1. 上游包 @mariozechner/pi-coding-agent 移除 SourceInfo 类型
2. TypeScript 编译失败
3. pnpm build 中断，dist/ 目录未更新
4. 插件运行时导入缺失的 normalizeLowercaseStringOrEmpty 函数
5. 19个插件全部加载失败
6. Agent无法回复任何消息

受影响的插件：memory-core、openai/anthropic/google、amazon-bedrock、ollama、browser/codex等19个。

未受影响的插件：飞书文档、飞书聊天、飞书知识库、飞书云盘、微信。所以渠道能收消息，但agent框架挂了。

临时修复

kill -HUP <PID>：给Gateway进程发送SIGHUP信号，重载配置但不重新构建dist。飞书和微信恢复回复。

最终修复

rm extensions/openclaw.plugin.json
git checkout -- .
git clean -fd
openclaw gateway start
注意：git clean -fd 会删除不在git追踪中的文件（自定义skills等），操作前一定要先备份！

诊断命令大全

# 查看实时日志
openclaw logs --follow

# 检查Gateway状态
openclaw gateway status

# 检查插件状态
openclaw plugins list

# 备份配置（升级前必做！）
cp ~/.openclaw/openclaw.json ~/.openclaw/openclaw.json.bak.$(date +%Y%m%d)

经验教训

教训说明
升级前必做备份config + git stash -u
不要伪造manifest直接git clean -fd更干净
插件配置定期清理失效条目要及时删除
升级命令优先用gitgit pull origin main比openclaw update更可靠
遇到问题看日志openclaw logs --follow是第一步
