OpenClaw定时任务推送失败排查指南

📝 文章简介

配置了多个Channel的OpenClaw用户，可能会遇到定时任务突然报错"channel not configured"的问题。本文提供完整的诊断思路和解决方案，帮助你快速定位问题根因并修复。

关键词：OpenClaw、定时任务、Cron、Channel配置、推送失败

🔗 相关链接： AI工具导航站 Vercel镜像

一、问题现象

某天早上，你发现每天准时推送的简报没收到了。

查后台，一堆定时任务状态显示 error，错误信息却很奇怪：

Error: weixin not configured: please run `openclaw channels login --channel openclaw-weixin`

⚠️ 困惑点：你用的是飞书，从来没配过微信，为什么会报"微信未配置"？

二、症状识别

如果遇到以下症状，本文可能对你有帮助：

症状检查方法
任务状态显示 erroropenclaw cron list
错误信息提到未配置的channel查看日志或任务详情
部分任务正常，部分报错对比正常/报错任务的配置
报错集中在 isolated 任务检查 sessionTarget 字段

三、诊断思路

第一步：看历史，找转折点

查看任务运行历史，找出从正常变异常的时间点：

# 查看任务运行历史
openclaw cron runs [jobId]
日期状态说明
4月1日-4月5日✅ ok正常
4月6日❌ error← 转折点
4月7日-4月8日❌ error持续报错

第二步：看配置，找空账号

检查 channels 配置，看是否有账号为空的channel：

# 查看配置文件中的channels
cat ~/.openclaw/openclaw.json | jq '.channels'

发现问题：

{
  "channels": {
    "feishu": { "accounts": { "main": {...} } },
    "openclaw-weixin": { "accounts": {} }  // ← 空的！
  }
}

第三步：看 sessionTarget，找差异

任务sessionTarget状态
AI导航站每日提醒session:agent:main:main✅ 正常
每日简报推送isolated❌ 报错

四、问题机制

📌 报错机制： Cron任务在 isolated session 运行 → 没有channel上下文 → 系统检测到多个channel → delivery模块默认选择 → 选到空账号channel → 💥报错

五、解决方案

方案一：明确指定 delivery.channel（推荐）

openclaw cron update [jobId] --patch '{"delivery":{"channel":"feishu"}}'

方案二：禁用未使用的 channel

编辑配置文件，将 plugins.entries.openclaw-weixin.enabled 设为 false。

六、验证修复

检查项预期结果命令
任务状态enabledopenclaw cron list
运行记录deliveredopenclaw cron runs [jobId]

七、最佳实践

  • 明确指定 delivery.channel
  • 禁用不用的 channel 插件
  • 定期检查任务状态

八、总结

🎯 一句话结论
多 channel 配置时，isolated 任务必须明确指定 delivery.channel。
