一、400错误的真正原因

OpenClaw配置飞书BOT时，400错误通常不是配置参数错误，而是飞书开放平台权限未开通。

⚠️ 为什么官方文档没写清楚？

飞书开放平台的权限配置分散在多个页面，有些权限名称和实际用途不一致，容易遗漏。

400错误的本质：飞书BOT调用API时，发现你缺少某个权限，直接拒绝请求。但错误信息只显示"400"，不告诉你具体缺哪个权限。

「400错误的真相：不是配置错，而是权限漏。」

二、检查清单：你的配置是否完整

先检查这5项基础配置，确认无误后再看隐藏权限：

  • App ID 正确复制
  • App Secret 正确复制
  • Encrypt Key 正确复制（飞书后台→事件订阅→加密策略）
  • Verification Token 正确复制
  • BOT已发布到飞书应用目录

如果以上5项都正确，但还是报400错误，继续往下看——你缺的是隐藏权限。

三、解决方案：开通3个隐藏权限

飞书开放平台有3个容易被忽略的权限，开通后400错误立即消失。

隐藏权限1：消息发送权限

飞书开放平台 → 应用详情 → 权限管理
搜索：contact:user.base:readonly
状态：已开通 ✅

这个权限让BOT能够读取用户基本信息，发送消息时需要用到。很多人只开通了"消息接收权限"，忘了开通"消息发送权限"。

隐藏权限2：群聊权限

飞书开放平台 → 应用详情 → 权限管理
搜索：im:chat:readonly
状态：已开通 ✅

如果BOT要发送消息到群聊，必须开通这个权限。

隐藏权限3：机器人权限

飞书开放平台 → 应用详情 → 权限管理
搜索：im:bot
状态：已开通 ✅

这是最容易被忽略的权限。BOT调用飞书API时，必须开通这个权限才能正常工作。

✅ 开通这3个权限后，重启OpenClaw Gateway：

openclaw gateway restart

等待10秒后，再次测试发送消息。

四、验证方法：5分钟确认修复成功

开通权限后，用这3步验证是否修复成功：

Step 1：重启Gateway

openclaw gateway restart

Step 2：检查Gateway状态

openclaw gateway status

确认状态显示：running

Step 3：测试发送消息

在飞书BOT对话框中发送一条消息，检查OpenClaw是否能正常接收和回复。

  • BOT能接收消息 ✅
  • BOT能回复消息 ✅
  • 没有400错误 ✅

如果以上3项都通过，恭喜你，踩坑成功上岸！

五、避坑总结：下次不再踩

「下次遇到400错误，先检查这3个隐藏权限。」

400错误排查流程：

  • 检查基础配置（App ID、App Secret等）
  • 检查消息发送权限（contact:user.base:readonly）
  • 检查群聊权限（im:chat:readonly）
  • 检查机器人权限（im:bot）
  • 重启Gateway并测试

https://open.feishu.cn/app/{App ID}/permission/manage

把你的App ID替换进去，直达权限管理页面。
