🚀 GitHub Pages + Vercel双平台部署指南

想让你的项目同时跑在GitHub Pages和Vercel上？这篇文章教你如何实现双平台部署，让网站更稳定、访问更快速。

相关链接： GitHub Pages演示 Vercel镜像

一、为什么要双平台部署？

单平台部署有风险：

平台潜在问题影响
GitHub Pages国内访问慢、偶尔不稳定用户体验差
Vercel免费额度有限、国内偶尔受限服务中断

双平台优势：

  • 互为备份：一个平台挂了，另一个还能访问
  • 访问加速：国内用户用Vercel，海外用户用GitHub
  • 完全免费：两个平台都是免费托管

二、前置条件

开始之前，确保你有：

  • GitHub账号
  • Vercel账号（可用GitHub登录）
  • 一个静态项目（HTML/CSS/JS）

三、GitHub Pages部署

3.1 创建仓库

# 创建新仓库
mkdir my-project
cd my-project
git init

# 创建首页
echo "<h1>Hello World</h1>" > index.html

# 推送到GitHub
git add .
git commit -m "Initial commit"
git remote add origin https://github.com/你的用户名/my-project.git
git push -u origin main

3.2 开启GitHub Pages

步骤： 1. 进入仓库Settings页面 2. 找到Pages选项 3. Source选择Deploy from a branch 4. Branch选择main，文件夹选择/(root) 5. 点击Save

等待几分钟，网站就会上线：

✅ 访问地址：https://你的用户名.github.io/my-project/

四、Vercel部署

4.1 导入GitHub仓库

步骤： 1. 登录 https://vercel.com 2. 点击"New Project" 3. 选择"Import Git Repository" 4. 找到你的GitHub仓库 5. 点击Import

4.2 配置项目

Vercel会自动检测项目类型：

项目类型自动检测默认配置
纯静态HTML✅ 自动无需配置
React/Vue✅ 自动自动npm build
Next.js✅ 自动优化配置

点击"Deploy"，等待部署完成。

✅ 访问地址：https://my-project-xxx.vercel.app/

五、自动化同步

现在每次push到GitHub，两个平台都会自动部署！

# 本地修改后
git add .
git commit -m "Update content"
git push

# GitHub Pages自动更新
# Vercel自动监听GitHub仓库，同步更新
⚠️ 注意：Vercel默认监听main分支，每次push都会触发自动部署。

六、自定义域名（可选）

6.1 GitHub Pages配置域名

在仓库根目录创建CNAME文件：

echo "yourdomain.com" > CNAME
git add CNAME
git commit -m "Add custom domain"
git push

6.2 Vercel配置域名

步骤： 1. Vercel项目设置 - Domains 2. 添加你的域名 3. 配置DNS解析（A记录或CNAME）

6.3 DNS配置示例

类型名称值
A@76.76.21.21（Vercel）
CNAMEwwwcname.vercel-dns.com

七、常见问题

Q1: 部署失败怎么办？

检查构建日志：

  • GitHub Pages: Actions页面查看日志
  • Vercel: Deployments页面查看日志

Q2: 国内访问哪个更快？

实测对比：

平台天津测试结果
GitHub Pages~2秒较慢
Vercel~0.5秒更快
💡 建议：国内用户优先访问Vercel链接，海外用户访问GitHub Pages。

Q3: 如何强制刷新部署？

# Vercel强制重新部署
vercel --prod --force

# GitHub Pages触发重新构建
git commit --allow-empty -m "Trigger rebuild"
git push

八、完整流程总结

步骤GitHub PagesVercel
1. 创建项目创建仓库导入仓库
2. 配置部署Settings > Pages自动检测
3. 自动同步push触发监听GitHub
4. 自定义域名CNAME文件Domains设置

九、实战案例

  • Vercel https://ai-tools-hub-zeta.vercel.app/

两个地址内容完全一致，互为备份！

十、总结

双平台部署的核心价值：

  • 稳定：单点故障不影响用户访问
  • 快速：不同地区自动选择最优节点
  • 免费：两个平台都是免费托管
  • 简单：一次push，两处更新
🎉 开始部署：照着这篇文章操作，5分钟就能让你的项目跑在两个平台上！
