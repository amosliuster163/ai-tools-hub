## 一、网站定位

### 1.1 目标用户
- AI工具使用者（开发者、产品经理、创业者）
- AI爱好者（想了解AI工具的普通用户）
- 内容创作者（寻找AI辅助工具）

### 1.2 网站价值
- **精选：** 不追求大而全，只推荐真正好用的工具
- **深度：** 每个工具都有详细评测和使用指南
- **洞察：** 结合人类痛点研究，提供市场机会分析

### 1.3 网站特色
- 中 Serif 字体风格（典雅、专业）
- 温暖配色（米白底色 + 深蓝主色 + 金色点缀）
- 内容专栏化（工具 + 洞察双主线）

## 二、设计风格

### 2.1 字体系统

| 用途 | 字体 | 来源 |
|------|------|------|
| 标题/品牌名 | Playfair Display | Google Fonts |
| 正文/描述 | Noto Sans SC + Source Serif Pro | Google Fonts |
| 卡片名称 | Source Serif Pro | Google Fonts |

**字体大小规范：**
```
网站标题：2.5rem - 3rem
分类标题：1.5rem - 1.8rem
卡片名称：1.1rem - 1.2rem
正文描述：0.95rem - 1rem
统计数据：0.85rem
```

### 2.2 颜色系统

| 颜色变量 | 色值 | 用途 |
|----------|------|------|
| --bg | #F5F0E8 | 页面背景（米白色） |
| --bg-card | #FFFFFF | 卡片背景 |
| --primary | #1e3a5f | 主色调（深蓝） |
| --secondary | #2d5a7b | 辅助色（中蓝） |
| --accent | #c9a227 | 点缀色（金色） |
| --text | #1a1a1a | 正文文字 |
| --muted | #6b7b8c | 辅助文字 |

**颜色使用规则：**
- 背景：米白色为主，深蓝用于Hero/Footer
- 文字：深色正文，灰色辅助说明
- 点缀：金色用于标签、分数、重点突出
- 卡片：白色背景，微阴影

### 2.3 阴影系统

```css
/* 柔和阴影（默认） */
box-shadow: 0 2px 12px rgba(30,58,95,0.08);

/* Hover阴影（交互反馈） */
box-shadow: 0 8px 24px rgba(30,58,95,0.12);
```

### 2.4 圆角规范

```
卡片外框：16px
按钮/标签：8px - 20px
图片容器：12px
```

## 四、内容规范

### 4.1 内容分类

| 分类ID | 名称 | 图标 | 说明 |
|--------|------|------|------|
| all | 全部内容 | - | 显示所有内容 |
| insight | 需求洞察 | 💡 | 痛点研究、市场分析 |
| model | 大模型 | 🤖 | AI模型服务 |
| image | 图像生成 | 🎨 | AI绘图工具 |
| video | 视频生成 | 🎬 | AI视频工具 |
| search | 搜索工具 | 🔍 | AI搜索引擎 |
| audio | 音频音乐 | 🎵 | AI音频工具 |
| doc | 文档处理 | 📄 | AI文档工具 |
| other | 其他 | 📦 | 其他AI工具 |

### 4.2 卡片规范

**首页卡片必须包含：**
```html
<article class="tool-card" data-category="分类ID">
    <div class="card-image-wrapper">
        <div class="card-placeholder">
            <span class="card-icon">图标emoji</span>
        </div>
    </div>
    <div class="card-content">
        <span class="tool-tag hot/new/热门">标签</span>
        <h4 class="tool-name">工具名称</h4>
        <p class="tool-desc">工具描述 · 特色功能 · 适用人群</p>
        <div class="tool-stats">
            <span class="stat-item">统计1</span>
            <span class="stat-item">统计2</span>
        </div>
    </div>
</article>
```

**卡片描述规范：**
- 使用 · 分隔多个要点
- 不超过3个要点
- 每个要点不超过8字

**统计数据规范：**
- 最多2个统计项
- 使用真实数据（不虚构）
- 如无数据可省略stats区域

### 4.3 标签规范

| 标签类型 | 样式 | 使用场景 |
|----------|------|----------|
| hot | 红色背景 | 热门工具、高优先级痛点 |
| new | 蓝色背景 | 新上线工具、新发现 |
| 默认 | 金色背景 | 普通标签 |

## 五、交互规范

### 5.1 分类筛选（v1.1更新）

**交互流程：**
1. 用户点击分类按钮
2. 按钮变为active状态（深蓝背景）
3. 卡片根据data-category筛选显示/隐藏
4. 需求洞察专栏动态显示/隐藏
5. 分隔符标题动态更新
6. 计数器更新显示数量

**分类名称映射表：**
```javascript
const categoryNames = {
  'all': '🤖 AI工具',
  'insight': '',           // 空字符串 → 隐藏分隔符
  'model': '🤖 大模型',
  'image': '🎨 图像生成',
  'video': '🎬 视频生成',
  'search': '🔍 搜索工具',
  'audio': '🎵 音频音乐',
  'doc': '📄 文档处理',
  'other': '📦 其他工具'
};
```

**完整筛选逻辑（v1.1）：**
```javascript
document.addEventListener('DOMContentLoaded', function() {
  document.querySelectorAll('.category-btn').forEach(btn => {
    btn.addEventListener('click', function() {
      const category = this.dataset.category;
      
      // 1. 筛选卡片
      cards.forEach(card => {
        card.style.display = 
          (category === 'all' || card.dataset.category === category) 
          ? 'block' : 'none';
      });
      
      // 2. 需求洞察专栏显示控制
      insightSection.style.display = 
        (category === 'all' || category === 'insight') 
        ? 'block' : 'none';
      
      // 3. 分隔符标题动态更新
      divider.textContent = categoryNames[category] || '';
      divider.style.display = categoryNames[category] ? 'block' : 'none';
      
      // 4. 更新计数
      sectionCount.textContent = `${visibleCount} 个内容`;
    });
  });
});
```

**特殊处理：**
- `insight`分类：分隔符标题为空字符串，自动隐藏
- 空分类：显示占位卡片+"筹备中"提示

### 5.2 卡片Hover

## 七、内容扩展规则

### 7.1 添加新工具

**步骤：**
1. 在index.html的tools-grid中添加新卡片
2. 设置正确的data-category
3. 创建对应的详情页（如有需要）
4. 更新section-count计数

**命名规范：**
- 详情页命名：`detail-{工具名英文}.html`
- 卡片图标使用emoji（不使用图片）

### 7.2 添加新分类

**步骤：**
1. 在分类导航添加新按钮
2. 设置data-category和图标emoji
3. 更新卡片data-category属性
4. 确保筛选脚本正确识别

### 7.3 添加新专栏

**步骤：**
1. 在index.html添加独立section
2. 使用insight-section样式（如需求洞察）
3. 创建对应详情页
4. 在分类导航添加筛选按钮
5. 详情页使用独立Hero设计
